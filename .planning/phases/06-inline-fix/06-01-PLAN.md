## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01-01 | Information Disclosure (XSS via item.text) | views/partials/grocery-item.njk + views/partials/recipe-ingredient-line.njk | mitigate | Nunjucks autoescape is on by default for these partials (no `\| safe` filter applied); user-supplied `item.text` and `ing.text` are escaped when interpolated into `aria-label="…"` and the `<span>` body. The `\| urlencode` filter on the `text=` query param escapes URL-special characters. Verified by Pitfall-4 test (`item.text = 'salt & pepper'` round-trip). |
| T-06-01-02 | Tampering (forged libraryEntryId in DOM) | views/partials/grocery-item.njk pencil hx-get | accept | Single-user LAN trust model (CLAUDE.md C-05). Even if a malicious local user forges a `libraryEntryId` query param, the server-side route in Plan 02 looks up the entry by id and 404s on no match — no privilege escalation possible. |
