<objective>
Add a per-recipe ingredient view toggle on the recipe detail page that flips the
Ingredients section between an "Original" pristine flat list (scraped order, no
category headers, no pencil controls) and the existing "Categorized" grouped view
(decorateIngredients output with category headers and pencil controls).

Purpose: Let the user see the untouched source ingredient list alongside the
curated/categorized view, persisting the preference per recipe.

Output: New mutation route POST /recipes/:id/view, three new partials, a
restructured recipe.njk Ingredients section, CSS, and tests. A new optional
recipe.viewMode field ('original' | 'processed'; absence means 'original').

CRITICAL CONSTRAINT (CLAUDE.md render-time categorization): store ONLY the UI
preference (viewMode). NEVER store computed categories. decorateIngredients() must
run fresh on every render in processed mode. Do NOT add viewMode to
defaultState/migrate; do NOT backfill existing recipes.
</objective>

         const { mondayOf } = require('../lib/week');
         const monday = mondayOf(new Date());
         const week = (state.weeks || []).find(w => w.weekStart === monday);
         const isTagged = !!(week && week.recipeIds.includes(recipe.id));
         const viewMode = recipe.viewMode === 'processed' ? 'processed' : 'original';
         return {
